
**Дата:** 19 ноября 2025 г.

**Статус:** Принято

**Контекст:** 

Для создания клиентского приложения сервиса парковки необходимо выбрать технологию, которая обеспечит кроссплатформенную разработку, высокую производительность, адаптивный интерфейс и нативный пользовательский опыт. Особое внимание уделяется работе с картами в реальном времени, push-уведомлениям и интеграции с платежными системами.

**Рассмотренные варианты:**

1. **React Native:** Популярный фреймворк для создания нативных мобильных приложений с использованием React.
2. **Flutter:** Кроссплатформенный инструментарий от Google с использованием языка Dart и собственного рендеринга.
3. **Нативная разработка:** Раздельная разработка для iOS (Swift) и Android (Kotlin).

**Решение:**

Выбран React Native.

**Обоснование:**

- **Кроссплатформенность:** Единая кодовая база для iOS и Android значительно ускоряет разработку и снижает затраты.
- **Нативный пользовательский опыт:** Доступ к нативным компонентам и API обеспечивает производительность, близкую к нативным приложениям.
- **Сильное сообщество:** Огромная экосистема библиотек и инструментов, включая готовые решения для карт, платежей и уведомлений.
- **Совместимость с существующими технологиями:** Возможность интеграции с выбранным бэкендом на Go через REST API и WebSockets.
- **Горячая перезагрузка:** Ускоряет процесс разработки и тестирования изменений.
- **Поддержка TypeScript:** Обеспечивает статическую типизацию для повышения надежности кода.

**Последствия:**

- Мобильные приложения для iOS и Android будут разрабатываться на React Native.
- Необходимо сформировать команду с опытом React Native или организовать обучение.
- Требуется выбрать и стандартизировать стэк дополнительных библиотек (навигация, состояние, карты, уведомления).
- Необходимо предусмотреть написание нативных модулей для специфичной функциональности, не покрытой существующими библиотеками.
- Следует внедрить строгие процессы code review и тестирования для поддержания качества кроссплатформенного приложения.
