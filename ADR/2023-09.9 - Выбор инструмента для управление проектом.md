**ADR: 2025-11.7 - Выбор инструмента для управления проектом**

**Дата:** 19 ноября 2025 г.

**Статус:** Принято

**Контекст:**

Для эффективного управления разработкой Сервиса Парковки в рамках выбранного процесса Scrumban необходим инструмент, который обеспечит визуализацию рабочего процесса, планирование спринтов, отслеживание прогресса и прозрачность для всех участников команды. Инструмент должен поддерживать распределенную команду и интеграцию с другими используемыми системами.

**Рассмотренные варианты:**

1. **Jira:** Мощная система для управления проектами с глубокой настройкой рабочих процессов
2. **Azure DevOps Boards:** Интегрированное решение от Microsoft с полным циклом разработки
3. **Linear:** Современный инструмент с фокусом на скорость и минималистичный интерфейс
4. **Trello:** Простая канбан-доска с базовыми функциями управления задачами
5. **GitHub Projects:** Интегрированное решение для команд, уже использующих GitHub

**Решение:**

Выбран Jira Software Cloud.

**Обоснование:**

- **Гибкость процессов:** Полная поддержка гибридного подхода Scrumban с возможностью настройки рабочих процессов
- **Мощная аналитика:** Встроенные отчеты и дашборды для отслеживания velocity, cycle time, burn-down charts
- **Интеграционная экосистема:** Широкие возможности интеграции с GitHub, Slack, Confluence, и другими инструментами стека
- **Масштабируемость:** Поддержка как отдельных команд, так и масштабирования на несколько команд в будущем
- **Готовые шаблоны:** Предопределенные шаблоны для Scrum и Kanban с возможностью кастомизации
- **Управление бэклогом:** Мощные инструменты для приоритизации и управления product backlog

**Почему не другие варианты:**

- **Azure DevOps Boards:** Сильная привязка к экосистеме Microsoft, менее гибкая для смешанных технологических стеков
- **Linear:** Менее развитые возможности для управления портфелем проектов и enterprise-требований
- **Trello:** Недостаточно функционала для полноценного управления спринтами и метриками
- **GitHub Projects:** Ограниченные возможности для кастомизации рабочих процессов и отчетности

**Последствия:**

- Необходимо настроить рабочие процессы в Jira в соответствии с выбранным процессом Scrumban
- Требуется интеграция Jira с GitHub для автоматического обновления статусов задач
- Необходимо обучение команды работе с Jira и используемым рабочим процессам
- Следует настроить дашборды и отчеты для отслеживания ключевых метрик проекта
- Требуется регулярный аудит и оптимизация рабочих процессов на основе обратной связи команды
- Необходимо определить роли и права доступа для различных участников проекта
- Следует настроить автоматические уведомления и интеграцию с Slack для коммуникации
